SQL(RDBMS) VS NoSQL

SQL мһҘм җ

  • кҙҖкі„нҳ• лҚ°мқҙн„°лІ мқҙмҠӨлҠ” лҚ°мқҙн„°лҘј нҡЁмңЁм Ғ, мІҙкі„м ҒмңјлЎң м ҖмһҘ л°Ҹ кҙҖлҰ¬к°Җ к°ҖлҠҘ
  • лҚ°мқҙн„° кө¬мЎ°лҘј м •мқҳн•ЁмңјлЎңмҚЁ лҚ°мқҙн„°мқҳ мҷ„м „м„ұмқҙ ліҙмһҘ
  • нҠёлһңмһӯм…ҳ
  • м •нҳ•нҷ”лҗң лҚ°мқҙн„°, лҚ°мқҙн„°мқҳ мҷ„м „м„ұмқҙ мӨ‘мҡ”н•ң лҚ°мқҙн„°лҘј м ҖмһҘн•ҳлҠ”лҚ° мң лҰ¬н•ҳлӢӨ.
  • ex) м „мһҗмғҒкұ°лһҳ м •ліҙ, мқҖн–ү кі„мўҢ м •ліҙ, кұ°лһҳ м •ліҙ л“ұ

SQL лӢЁм җ

  • н…Ңмқҙлё” кө¬мЎ° ліҖнҷ” л“ұм—җ лҚң мң м—°н•ҳлӢӨ.
  • нҷ•мһҘм„ұмқҙ лӮ®лӢӨ. (нҷ•мһҘн•ҳлҠ”лҚ° л№„мҡ©мқҙ л§Һмқҙ л“¬)

NoSQL мһҘм җ

  • лҚ°мқҙн„° кө¬мЎ°лҘј лҜёлҰ¬ м •мқҳн•ҳм§Җ м•Ҡм•„лҸ„ лҗЁмңјлЎң м ҖмһҘн•ҳлҠ” лҚ°мқҙн„°мқҳ кө¬мЎ° ліҖнҷ”м—җ мң м—°н•ҳлӢӨ.
  • нҷ•мһҘн•ҳкё° л№„көҗм Ғ мүҪлӢӨ.
  • л°©лҢҖн•ң м–‘мқҳ лҚ°мқҙн„°лҘј м ҖмһҘн•ҳлҠ”лҚ° мң лҰ¬н•ҳлӢӨ.
  • л№„м •нҳ•нҷ”лҗң лҚ°мқҙн„°лӮҳ мҷ„м „м„ұмқҙ мғҒлҢҖм ҒмңјлЎң лҚң мң лҰ¬н•ң лҚ°мқҙн„°лҘј м ҖмһҘн•ҳлҠ”лҚ° мң лҰ¬н•ҳлӢӨ.
  • ex) лЎңк·ёлҚ°мқҙн„°

NoSQL лӢЁм җ

  • лҚ°мқҙн„°мқҳ мҷ„м „м„ұмқҙ лҚң ліҙмһҘлҗңлӢӨ.
  • нҠёлһңмһӯм…ҳмқҙ м•Ҳлҗҳкұ°лӮҳ л№„көҗм Ғ л¶Ҳм•Ҳм •н•ҳлӢӨ.

  2. мһ‘м—… м „мҡ© лёҢлһңм№ҳм—җм„ң мһ‘м—…мқ„ м§„н–ү

н”„лЎңм қнҠём—җ мЎҙмһ¬н•ҳлҠ” лӘЁл“  лёҢлһңм№ҳлҘј нҷ•мқён•ҳкі  мӢ¶лӢӨл©ҙ м•„лһҳмқҳ лӘ…л №м–ҙлҘј мһ…л Ҙн•ҳл©ҙ лҗңлӢӨ.

2020-10-02Git9л¶„ мқҪкё° (лҢҖлһө 1375 лӢЁм–ҙ)

Git кё°ліё 1

GitмқҙлһҖ?

git кіөмӢқ нҷҲнҺҳмқҙм§Җм—җм„ң gitм—җ лҢҖн•ҙ м„ӨлӘ…н•ң лӮҙмҡ©мқҙлӢӨ.

Git кіөмӢқ нҷҲнҺҳмқҙм§Җ

Git is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency.

мүҪкІҢ л§җн•ҳл©ҙ н”„лЎңм қнҠё нҢҢмқјмқҳ ліҖкІҪ мӮ¬н•ӯмқ„ м¶”м Ғн•ҳлҠ” мӢңмҠӨн…ңмқёлҚ° мқҙлҹ¬н•ң Gitмқ„ нҶөн•ҙ к°ңл°ңмһҗл“ӨмқҖ н”„лЎңм қнҠёмқҳ ліҖкІҪ мӮ¬н•ӯмқ„ кё°лЎқн•ҳкі , нҠ№м • мӢңм җмқҳ лІ„м „мңјлЎң м–ём ңл“  лҸҢм•„к°Ҳ мҲҳ мһҲкІҢ лҗҳм—ҲлӢӨ.

мқҙлҹ¬н•ң GitмқҖ нҳ‘м—…н•  л•Ң лҚ”мҡұ нҡЁкіјк°Җ к·№лҢҖнҷ” лҗҳлҠ”лҚ° к°Ғк°Ғмқҳ к°ңл°ңмһҗк°Җ к°Ғкё° лӢӨлҘё н”„лЎңм қнҠё лІ„м „мқ„ ліёмқё м»ҙн“Ён„°м—җ к°–кІҢ н•ҳм—¬ к°ңл°ңмқ„ м§„н–үн•ң нӣ„ мқҙлҹ¬н•ң к°ңлі„ лІ„м „мқҳ н”„лЎңм қнҠёл“Өмқ„ Gitмқ„ нҶөн•ҙ лі‘н•©н•ҳм—¬ кё°мӨҖмқҙ лҗҳлҠ” лІ„м „мқҳ н”„лЎңм қнҠём—җ м Ғмҡ©н•ҳлҠ” кІғмқҙ мүҪкІҢ к°ҖлҠҘн•ҙмЎҢлӢӨ.

мҰү, Gitмқ„ мӮ¬мҡ©н•ЁмңјлЎңмҚЁ мІҙкі„м Ғмқё к°ңл°ңмқҙ к°ҖлҠҘн•ҙм§ҖкІҢ лҗҳм—ҲлӢӨ.

Git м„Өм№ҳн•ҳкё°

Git лӢӨмҡҙлЎңл“ң л§ҒнҒ¬

мһҗмӢ мқҙ мӮ¬мҡ©н•ҳлҠ” OSм—җ л§һлҠ” кІғмқ„ м„ нғқн•ҙ м„Өм№ҳн•ҳл©ҙ лҗңлӢӨ.
м„Өм№ҳлҘј л§ҲміӨлӢӨл©ҙ н„°лҜёл„җ нҳ№мқҖ CMDм—җм„ң м•„лһҳмқҳ м»Өл§Ёл“ңлҘј мһ…л Ҙн•ҙліҙмһҗ.

Gitмқҙ м •мғҒм ҒмңјлЎң м„Өм№ҳлҗҳм—ҲлӢӨл©ҙ, м„Өм№ҳлҗҳм–ҙ мһҲлҠ” Git лІ„м „мқҙ н‘ңмӢңлҗңлӢӨ.

Repositories(м ҖмһҘмҶҢ)

Gitм—җм„ңлҠ” repositoryлқјлҠ” мҡ©м–ҙк°Җ мһҲлӢӨ.
Git repositoryлҠ” GitмңјлЎң кҙҖлҰ¬н•ҳлҠ” н”„лЎңм қнҠё м ҖмһҘмҶҢмқҙлӢӨ.
н”„лЎңм қнҠёлҘј м ҖмһҘн•ҳлҠ” н•ҳлӮҳмқҳ нҸҙлҚ”лқјкі  ліҙл©ҙ лҗңлӢӨ.

Git repositoryлҠ” нҒ¬кІҢ л‘җ к°Җм§Җ мў…лҘҳк°Җ мһҲлӢӨ.

  • Local repository : ліёмқёмқҳ м»ҙн“Ён„°м—җ м ҖмһҘлҗң лЎңм»¬ лІ„м „мқҳ н”„лЎңм қнҠё м ҖмһҘмҶҢ
  • Remote repository : мҷёл¶Җ(мӣҗкІ© м„ңлІ„)м—җ мһҲлҠ” мҷёл¶Җ лІ„м „мқҳ н”„лЎңм қнҠё м ҖмһҘмҶҢ

ліҙнҶө Local repositoryм—җ Remote repositoryмқҳ н”„лЎңм қнҠёлҘј к°Җм ёмҷҖм„ң мҲҳм • л°Ҹ к°ңл°ңмқ„ н•ҳкі  Remote repositoryм—җ лЎңм»¬м—җм„ң мһ‘м—…н•ң мӮ¬н•ӯл“Өмқ„ лі‘н•©н•ҳкі  ліҖкІҪ мӮ¬н•ӯмқ„ м Ғмҡ©н•ңлӢӨ.

кІ°көӯ лҚ°мқҙн„°лІ мқҙмҠӨлҠ” лҚ°мқҙн„°л“Өмқ„ м•Ҳм „н•ҳкі  мІҙкі„м ҒмңјлЎң м ҖмһҘн•ҳкі  ліҙмЎҙн•ҳкё° мң„н•ҙ мғқк°Ғн•ҙлӮё л°©лІ•мқҙлӢӨ.

кҙҖкі„нҳ• лҚ°мқҙн„°лІ мқҙмҠӨ(RDBMS, Relational DataBase Management System)

кҙҖкі„нҳ• лҚ°мқҙн„° лӘЁлҚём—җ кё°мҙҲлҘј л‘” лҚ°мқҙн„°лІ мқҙмҠӨ мӢңмҠӨн…ң
кҙҖкі„нҳ• лҚ°мқҙн„°лһҖ лҚ°мқҙн„°лҘј м„ңлЎң мғҒнҳёкҙҖл Ём„ұмқ„ к°Җм§„ нҳ•нғңлЎң н‘ңнҳ„н•ң лҚ°мқҙн„°лҘј л§җн•Ё
ex) MySQL, Postgres, Oracle DB

кҙҖкі„нҳ• лҚ°мқҙн„°лІ мқҙмҠӨмқҳ нҠ№м§•

  • лӘЁл“  лҚ°мқҙн„°л“ӨмқҖ 2м°Ёмӣҗ н…Ңмқҙлё”лЎң н‘ңнҳ„
  • к°Ғк°Ғмқҳ н…Ңмқҙлё”мқҖ м„ңлЎң мғҒнҳёкҙҖл Ём„ұмқ„ к°Җм§Җкі  м„ңлЎң м—°кІ°лҗ  мҲҳ мһҲлӢӨ.
  • к°Ғк°Ғмқҳ н…Ңмқҙлё”мқ„ м»¬лҹј(column)кіј row(лЎңмҡ°)лЎң кө¬м„ұ
  • м»¬лҹјмқҖ н…Ңмқҙлё”мқҳ к°Ғ н•ӯлӘ©мқ„ л§җн•ңлӢӨ.
  • лЎңмҡ°лҠ” к°Ғ н•ӯлӘ©л“Өмқҳ мӢӨм ң к°’мқ„ л§җн•ңлӢӨ.
  • к°Ғ лЎңмҡ°лҠ” м Җл§Ңмқҳ кі мң  нӮӨ(Primary Key)к°Җ мһҲлӢӨ.

н…Ңмқҙлё” к°„мқҳ м—°кІ°

нҒ¬кІҢ 3к°Җм§Җмқҳ мў…лҘҳк°Җ мһҲлӢӨ.

  1. one to one
  2. one to many
  3. many to many

1. One to One

  • н…Ңмқҙлё” Aмқҳ лЎңмҡ°мҷҖ н…Ңмқҙлё” Bмқҳ лЎңмҡ°к°Җ м •нҷ•нһҲ 1:1 л§Өм№ӯмқҙ лҗҳлҠ” кҙҖкі„



2. One to Many

  • н…Ңмқҙлё” Aмқҳ лЎңмҡ°к°Җ н…Ңмқҙлё” Bмқҳ м—¬лҹ¬ лЎңмҡ°мҷҖ м—°кІ°мқҙ лҗҳлҠ” кҙҖкі„
  • 1:N кҙҖкі„



3. Many to Many

  • н…Ңмқҙлё” Aмқҳ м—¬лҹ¬ лЎңмҡ°к°Җ н…Ңмқҙлё” Bмқҳ м—¬лҹ¬ лЎңмҡ°мҷҖ м—°кІ°мқҙ лҗҳлҠ” кҙҖкі„
  • N:N кҙҖкі„



н…Ңмқҙлё”мқ„ м—°кІ°н•ҳлҠ” л°©лІ•

  • нҠ№м • н…Ңмқҙлё”мқҳ кі мң  нӮӨк°Җ лҗҳлҠ” лЎңмҡ°лҘј Foreign key(мҷёл¶ҖнӮӨ)лЎң мӮ¬мҡ©н•ҳм—¬ мЈјлЎң м—°кІ°н•ңлӢӨ.
  • м•һмқҳ one to one мҳҲм—җм„ң user_profiles н…Ңмқҙлё”мқҳ user_id м»¬лҹјмқҖ user н…Ңмқҙлё”м—җ кұёл ӨмһҲлҠ” мҷёл¶Җ нӮӨлЎң м§Җм •н•ңлӢӨ.
  • мҰү, user_idмқҳ к°’мқҖ users н…Ңмқҙлё”мқҳ id к°’мқҙл©° user_id к°’мқҖ л¬ҙмЎ°кұҙ users н…Ңмқҙлё”мқҳ id м»¬лҹјм—җ мЎҙмһ¬н•ҳлҠ” к°’л§Ң мғқм„ұлҗ  мҲҳ мһҲлӢӨ.
  • л§Ңм•Ҫ users н…Ңмқҙлё”м—җ м—ҶлҠ” id к°’мқҙ user_idм—җ м§Җм •лҗҳл©ҙ м—җлҹ¬к°Җ лӮңлӢӨ.

н…Ңмқҙлё”мқ„ м—°кІ°н•ҳлҠ” мқҙмң 

  • н•ҳлӮҳмқҳ н…Ңмқҙлё”м—җ лӘЁл“  м •ліҙлҘј лӢӨ л„Јмңјл©ҙ лҸҷмқјн•ң м •ліҙл“Өмқҙ л¶Ҳн•„мҡ”н•ҳкІҢ мӨ‘ліөлҗҳм–ҙ м ҖмһҘлҗҳкё° л•Ңл¬ёмқҙлӢӨ.
  • мқҙлЎң мқён•ҙ лҚ” л§ҺмқҖ л””мҠӨнҒ¬(м ҖмһҘкіөк°„)лҘј мӮ¬мҡ©н•ҳкІҢ лҗңлӢӨ.
  • мһҳлӘ»лҗң лҚ°мқҙн„°к°Җ м ҖмһҘлҗ  к°ҖлҠҘм„ұмқҙ лҶ’м•„м§„лӢӨ.

н…Ңмқҙлё”мқ„ м—°кІ°н•ҳл©ҙ мң„мқҳ л¬ём ңк°Җ н•ҙкІ°лҗңлӢӨ.

мқҙл ҮкІҢ м—¬лҹ¬ н…Ңмқҙлё”м—җ лӮҳлҲ„м–ҙм„ң м ҖмһҘн•ҳлҠ” кІғмқ„ normalization(м •к·ңнҷ”)лқјкі  н•ңлӢӨ.
